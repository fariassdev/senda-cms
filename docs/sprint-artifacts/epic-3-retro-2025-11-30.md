## Challenges and Learnings

### Challenge: Story 3-5 Reorientation

**Initial approach:** Auto-save on every drag interaction

- Problem: Too many API calls, no undo capability

**Reoriented to:** Local state with explicit "Save Changes" button

- Solution: `pendingOrder` tracks local changes
- "Save Changes" button appears when unsaved changes exist
- "Unsaved Changes" modal on navigation
- Single API call on save

**Key Insight:**

> Prefer local state with explicit confirmation over auto-save for operations the user may want to review or undo.
